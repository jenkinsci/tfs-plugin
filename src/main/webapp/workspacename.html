The name of the Workspace under which the source should be retrieved. This workspace is created at the start of a download, and deleted at the end. You can normally omit the property unless you want to name a workspace to avoid conflicts on the server (i.e. when you have multiple projects on one server talking to a Team Foundation Server)

The default value is to create a workspace named Hudson-${JOB_NAME}. The TFS plugin supports the following macros that are replaced in the workspace name:

  • ${JOB_NAME} - The name of the job.
  • ${USER_NAME} - The user name that the Hudson server or slave is running as.
  • ${NODE_NAME} - The name of the node/slave that the plugin currently is executed on. Note that this is not the hostname, this value is Hudson configured name of the slave/node.
  • ${ENV} - The environment variable that is set on the master or slave.
